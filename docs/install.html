You can install surfcomp via three different ways: You can

  1. use a distribution specific installation package,
  2. create an installation package from the source packages or
  3. compile and install from the source tarballs.
The source tarballs provide the greatest flexibility but demand the greatest care and skills. The first option is more fail-proof but maybe not available for your Linux distribution. In that case you have to use option 2 first to create an RPM package set for your specific setup.

This page also explains how to register the Sybyl module of SURFCOMP.

Information about the installation of the surfcomp-monitor program can be found at the end of this page.

Distribution RPM packages

You must have installed all the required external packages for your system and you must be logged in as root.

  1. Unpack the tarball for your system into any empty directory
  2. Then change into that directory and call the script install.sh.

Thats all, the program suite is now ready to use.

Source RPM packages

You must have installed all the required external packages for your system and you must be logged in as root.

  1. Unpack the tarball for your system into any empty directory
  2. Then change into that directory and call the script rebuild.sh DESTDIR. DESTDIR is the path where the binary RPMs for your system should be placed.
  3. If all sources could be compiled successfully you can find a complete installation set for your distribution under DESTDIR including a README and the install script.
  4. Process with the steps described for the distribution RPM packages

If you think that the packages for your distribution are useful for other users I would appreciate if you send us the files and we put it on the webpage.

Compile and Install from Source Tarballs

The source files are organized by autoconf and automake. You can therefore compile and install each package by the following command sequence:

% gunzip PACKAGE.tar.gz
% tar fx PACKAGE.tar
% cd PACKAGE
% ./configure
% make
% make install

In order to avoid large specifications of the locations of all required packages we recommend to compile and immediately install the packages in the following order:

  1. RazorBack, bestrot, graph-algorithms, xmlutils
  2. libsurface, trimesh
  3. libsurfcomp
  4. surfcomp, surfcomp-control, surfcomp-monitor
(Packages under the same number can be installed in any arbitrary sequence.) Please refer to the INSTALL files of these packages to learn about the special flags and additional build possibilities.

Register the Sybyl module

To register the Sybyl module add the following lines to the .sybylrc file in your home directory:

setvar DUCKHOME PREFIX/share/surfcomp-control/sybyl

uims load $DUCKHOME/active_site.spl
uims load $DUCKHOME/duckCompare.spl
where PREFIX is the installation prefix of the surfcomp-control packages (usually /usr).

Install surfcomp-monitor

For the installation of the surfcomp-monitor you have the same choices as for the rest of the program suite: binary and source RPMs and a source-code tarball. Please look into the instructions above to learn how to use each of these options. In addition to this you have to install the surfcomp-monitor as a plug-in for the Geomview application.

To install the surfcomp-monitor as a plug-in of Geomview you have to create an appropriate entry in the private or system-wide configuration file of geomview. Add the line

(emodule-define "surfcomp-monitor" "surfcomp-monitor MYSQL_SERVER MYSQL_USER)
either to the file
  • .geomview in your home directory or to
  • .geomview in Geomview's system directory (/usr/share/geomview on my system).
MYSQL_SERVER and MYSQL_USER are the name of the MySQL server and MySQL user which are used to access the experiment databases. Further information about the installation of plug-in modules in Geomview can be found in the Geomview manual.
